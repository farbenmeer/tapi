## The stack

Toapi is a small family of composable packages. Start with `@toapi/server` to define your API, generate a typed client with `@toapi/client`, then add caching, routing, React bindings and a Vite plugin as you need them.

<CardGrid>
  <LinkCard title="@toapi/server" href="/tapi/server/" description="Define routes and handlers — the single source of truth for your API's types." />
  <LinkCard title="@toapi/client" href="/tapi/client/" description="An end-to-end typed fetch client inferred from your server definition." />
  <LinkCard title="@toapi/worker" href="/tapi/worker/" description="A service worker runtime that caches and revalidates API responses offline." />
  <LinkCard title="@toapi/common" href="/tapi/common/" description="Shared primitives: HttpError, TRequest/TResponse and the route type contract." />
  <LinkCard title="@toapi/cache" href="/tapi/cache/" description="Tag-based server cache with in-memory, filesystem, Redis and Postgres backends." />
  <LinkCard title="@toapi/router" href="/tapi/router/" description="A tiny, typed client-side router for React." />
  <LinkCard title="@toapi/react" href="/tapi/react/" description="React bindings — the useQuery hook with Suspense and live revalidation." />
  <LinkCard title="@toapi/vite-plugin" href="/tapi/vite-plugin/" description="Serve your API in dev and build a deployable server bundle with Vite." />
</CardGrid>

## Get started

```bash
pnpm add @toapi/server @toapi/client
```

Then head to the [server introduction](/tapi/server/) to define your first API.
